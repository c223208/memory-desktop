## 概要
本プロジェクトは、ブラウザ上でデスクトップ環境を模した操作体験を提供するwebアプリケーションである。  
複数ウィンドウの生成・移動・管理を可能とし、視覚的再現に留まらず、操作感と構造設計を重視している。

---

## 背景・コンセプト
本制作は、ゼミでお世話になった教授へ感謝の気持ちを伝えることを目的として開始した。  
文書を直接表示するのではなく、あえて操作の手間を介在させ、デスクトップ風の環境の中で手紙を「見つける」構造を採用している。そのようにして、探索・発見する楽しさを生むことを意図した。

## システム設計・構成

### 設計方針
- 各ウィンドウを独立したオブジェクトとして扱う
- 位置・サイズ・前後関係などの状態を一元管理する
- UI描画とロジック処理を分離し、保守性と拡張性を確保する

### 構成概要

```
/
├─ index.html   // デスクトップ画面のエントリポイント
├─ style.css    // デスクトップ全体およびウィンドウ共通スタイル
├─ script/
│ ├─ main.js    //初期化・基盤処理
│ ├─ config.js  // ウィンドウ・デスクトップ定義（配列管理）
│ ├─ ui.js      // UIイベント処理・操作ロジック
│ └─ boot.js    // BIOS風ブート画面の表示制御
├─ pages/       // iframeで読み込まれる各ウィンドウコンテンツ
├─ img/
│ └─ icon/      // デスクトップアイコン画像
└─ README.md

```

## 機能仕様
- デスクトップ画面の表示
- ウィンドウの生成・移動・重なり制御
- 最小化・最大化操作
- ウィンドウの状態をタスクバーに反映
- iframe / 文書コンテンツの表示
- 読み取り専用画像の表示
- 現在時刻の表示
- BIOSブート画面の表示

---

## 非機能要件
- 保守性：機能単位でのJS分割
- 保守性：イベント処理をJS側に集約し、DOM構造への依存を最小化
- 拡張性：新規ウィンドウ種別の追加が容易
- 操作性：直感的なマウス操作を前提
- 操作性：モバイルデバイスにも対応

## 開発プロセス
本プロジェクトは、アジャイル開発を採用し、短いサイクルで実装と調整を繰り返しながら進行した。 大まかな方針や機能仕様は両名で相談のうえ決定し、必要に応じて随時ミーティングを行った。  
一方で、各自が裁量を持って独立した機能・非機能要件を実装する場面も多く、実装後に相互レビューと調整を行う形で統合している。 コーディングにあたっては、両名ともAIツールを補助的に利用しつつ、設計判断および最終的な実装内容は人間が決定した。  
全体として、  
操作感・起動時の印象を重視した実装（[木下](https://github.com/c223083m)）  
保守性・拡張性を重視した構成改善（[三和](https://github.com/c223208)）  
という異なる関心領域を相互に補完し合う形で、システム全体を完成させた。
